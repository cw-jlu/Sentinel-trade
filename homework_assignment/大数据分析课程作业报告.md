# 大数据分析课程作业报告

**项目背景：**
基于真实的金融大数据流处理系统 (Sentinel Trade) 业务场景，对加密货币交易所产生的逐笔交易（Tick Data）进行清洗、聚合与价值发现。

## 一、 数据集总结

- **数据集来源**：金融市场的实时与历史逐笔交易数据（Tick Data）。
- **数据特征**：
  - **字段**：`Symbol`（交易对）、`Price`（成交价）、`Volume`（成交量）、`Timestamp`（时间戳）、`Trade_ID`（交易唯一标识）。
  - **体量与特性**：吞吐量极大，属于典型的高频时间序列流数据。存在网络延迟导致的乱序数据以及重复数据。
- **业务目的**：
  1. **实时计算**：将海量杂乱的逐笔交易，降维聚合压缩成标准的 K线数据 (OHLCV)。
  2. **离线分析**：对历史全量数据进行统计，计算各交易对的 VWAP (成交量加权平均价)，以发现大资金主力的实际建仓成本和市场趋势。

## 二、 处理方法与核心代码实现

本作业结合了实时流处理（Apache Flink）和离线批处理（Hadoop MapReduce）两种大数据处理方法。

### 1. 实时流处理 (Apache Flink)
使用 Flink 对高频流数据进行清洗、基于 `Trade_ID` 去重，并通过时间窗口（Time Window）进行滚动聚合计算 K 线。

**核心代码 (KLineAggregatorComplete)**：处理乱序并聚合开高低收价格。
```java
// KLineAggregatorComplete.java 节选
@Override
public KLineAccumulator add(TickData tick, KLineAccumulator acc) {
    acc.symbol = tick.getSymbol();
    // 更新开盘价（根据时间戳决定）
    if (tick.getTimestamp() < acc.minTimestamp) {
        acc.minTimestamp = tick.getTimestamp();
        acc.open = tick.getPrice();
    }
    // 更新收盘价
    if (tick.getTimestamp() > acc.maxTimestamp) {
        acc.maxTimestamp = tick.getTimestamp();
        acc.close = tick.getPrice();
    }
    // 更新最高/最低价和总成交量
    acc.high = Math.max(acc.high, tick.getPrice());
    acc.low = Math.min(acc.low, tick.getPrice());
    acc.volume += tick.getVolume();
    return acc;
}
```

### 2. 离线批处理 (Hadoop MapReduce)
使用 MapReduce 对沉淀的海量历史 CSV 日志进行批处理计算 VWAP。Map 阶段提取 Symbol 和量价，Reduce 阶段计算总额与加权平均。

**核心代码 (VWAPReducer)**：
```java
// VWAPReducer.java 节选
@Override
protected void reduce(Text key, Iterable<Text> values, Context context) throws IOException, InterruptedException {
    double totalTurnover = 0.0;
    double totalVolume = 0.0;

    for (Text val : values) {
        String[] pv = val.toString().split(",");
        double price = Double.parseDouble(pv[0]);
        double volume = Double.parseDouble(pv[1]);
        totalTurnover += (price * volume);
        totalVolume += volume;
    }

    if (totalVolume > 0) {
        double vwap = totalTurnover / totalVolume; // 计算成交量加权平均价
        String outputStr = String.format("Total Volume: %.2f, VWAP: %.4f", totalVolume, vwap);
        context.write(key, new Text(outputStr));
    }
}
```

## 三、 分析结论与业务洞察

基于上述 MapReduce 和 Flink 产出的统计结果，得出以下**统计学结论与商业洞察**：

1. **降维与性能优化**：通过 Flink 流式聚合，将每秒上万条的 Tick 数据压缩为每分钟一条的 K 线数据，数据量下降 99% 以上，大幅降低了下游 ClickHouse 的存储成本并提升了前端页面渲染的流畅度。
2. **趋势发现 (VWAP 现象)**：VWAP 是判断市场多空趋势及大资金动向的关键统计指标。当某一交易对的当前算术平均价格显著高于其 VWAP 时，说明大量筹码是在较低的价格区间成交的，少部分的高价交易拉升了当前价格。这在统计分布上呈现**右偏（正偏态）**的成交量分布。
3. **业务指导意义**：这种统计学特征表明市场处于强势上涨趋势，前期介入的大资金目前处于盈利状态，支撑力度较强；系统可据此发出上行突破的交易告警 (Alert)，指导后续的量化交易策略。
